# Clo Workflow

Use this skill as the top-level reference for the Codex adaptation of clo-author.

1. Read `references/quick-ref.md` for the pipeline and quality gates.
2. Read `references/domain-profile.md` and `references/journal-profiles.md` for the active field and journal calibration layer.
3. Read `references/explorations.md` when you want the upstream-style repo sandbox for prototypes and side investigations.
4. Read `references/translation-map.md` when you need to know where an upstream Clo-Author asset landed in Codex.
5. Read `references/source-rules/`, `references/source-references/`, and `references/source-explorations/` only when upstream provenance matters.
6. Keep the main Codex session as the orchestrator. Use the installed custom subagents and `$clo-*` skills rather than trying to recreate hidden upstream behavior.
7. Prefer repo-local rules and skills when they conflict with the generic workflow.
